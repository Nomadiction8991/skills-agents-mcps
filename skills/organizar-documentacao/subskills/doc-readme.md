# @doc-readme

## Papel
Manter `README.md` da raiz como mapa curto de navegação.

## Regras
- README aponta para `./docs`; não duplica conteúdo detalhado.
- Atualizar README quando surgir domínio novo, subtema central ou mudança importante de entrada.
- Limitar README a visão geral, links e instruções iniciais.
- Se arquivos forem movidos em `./docs`, corrigir links do README quando necessário.

## Estrutura sugerida
- visão geral do projeto;
- como começar;
- links para `./docs`;
- changelog ou releases.
