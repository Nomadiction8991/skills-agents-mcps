# @doc-fix

## Papel
Corrigir a documentação no ponto exato em que ficou errada ou incompleta.

## Regras
- Fix não cria arquivo novo.
- Corrigir descrição, tabela, exemplo ou parâmetro no arquivo existente.
- Escrever apenas o comportamento atual e correto.
- Não deixar nota de histórico no corpo da documentação.
- Registrar o histórico no changelog quando o bug tiver impacto externo.
- Se a inconsistência vier de markdown legado em lugar errado, corrigir e mover via `@doc-legado`.
